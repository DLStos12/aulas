Eventos DOM

Só pra recapitular, o DOM significa: Document Object Model, que são os elementos dentro do site, sem ele não conseguiriamos acessar os objetos da pagina.
e para acessar o DOM pode ser por ID, TagName, Name, Class ou QuerySelector.
agora vamos aos Eventos DOM.
Evento é tudo o que podemos fazer com essa DIV e o mais comum é Evento de mouse, por exemplo:
quando colocamos o mouse dentro da div, temos o evento mouseenter, a partir disso ao mover o mouse, temos o evento mousemove, ao fazer um click longo, ou seja, clicar e manter pressionado temos o mousedown e ao soltar o click, temos o mouseup, se apenas clicar temos o click e ao tirar o mouse da div, temos o mouseout, mas temos diversos outros eventos possiveis, inclusive tendo diferenciações de toutch de celular e clicks de mouse Observe esse site para ver os diversos eventos possiveis

Mas para dispararmos um evento devemos usar uma função, que é uma serie de codigos que serão acionados ao iniciar os eventos, por exemplo, se tivermos 10 linhas de codigo, teremos um bloco que não será executado automaticamente, diferente do que fizemos até agora com os alerts e prompt, eles só serão executados quando o usuario interagir com o comando e para criarmos uma finção deve seguir algumas regras:
o bloco de codigo deve estar dentro de uma function{bloco}, existem funções sem nome, como este que está lado, mas geralmente elas são nomeadas, com nomes da ação que será executada ficando assim: function ação(){bloco} e nas funções com nome podemos colocar parametros na função com nome function ação(parametro){boco}. observe a imagem com os codigos de exemplo:
